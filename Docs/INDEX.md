## 📋 Development Squad

The squad is the primary development workflow for Blaze.LlmGateway features. It provides:
- ✅ 9 specialized agents (Conductor, Planner, Architect, Coder, Tester, Reviewer, Infra, Security-Review, Orchestrator)
- ✅ 11 coordination skills (reasoning, quality-gate, claude-devfleet, team-builder, etc.)
- ✅ Two paths: phased (human-gated) or parallel (autonomous)
- ✅ Automatic guardrails (MEAI law, streaming, quality gates, security audits)

### Commands

| Command | Purpose | Use When |
|---------|---------|----------|
| `/agent squad "task"` | Start human-gated phased squad run | High-risk, complex features |
| `/orchestrate "task"` | Start autonomous parallel squad run | Clear, decomposable tasks |
| `/agent squad.planner "task"` | Research + spec + plan only (no implementation) | Planning phase |
| `/agent squad.architect` | Request an ADR for a specific decision | Architecture questions |
| `/agent squad.coder` | Direct code implementation (within squad context) | Implementation work |

### Architecture Decisions (ADRs)

- **[ADR-0010: Parallel-Orchestration Path](design/adr/0010-parallel-orchestration-path.md)** — Why the squad has two paths (phased vs. autonomous parallel)

### Squad Runs

All squad executions create timestamped run directories in `Docs/squad/runs/<timestamp>-<task-slug>/`:
- `reasoning.log.md` — Append-only decision trail
- `handoff/` — Inter-agent communication envelopes
- `spec.md` — Feature specification (from Planner)
- `plan.md` — Implementation plan (from Planner)
- `prd.md` — Product requirements (generated by Orchestrator for parallel path)

## 🎯 Running the Squad

### 1. Start a squad run
```bash
# Phased (recommended for first time)
/agent squad "Add resilient failover for provider timeouts"

# Parallel (once comfortable)
/orchestrate "Implement provider health checks in AppHost"
```

### 2. Monitor the run
```bash
# Watch the reasoning log
cat Docs/squad/runs/<timestamp>-<task-slug>/reasoning.log.md

# Check handoff envelopes (inter-agent communication)
ls Docs/squad/runs/<timestamp>-<task-slug>/handoff/

# For parallel runs, monitor worktrees
ls -la .worktrees/
```

### 3. Review phase outputs
```bash
# Read the spec and plan (Planner output)
cat Docs/squad/runs/<timestamp>-<task-slug>/spec.md
cat Docs/squad/runs/<timestamp>-<task-slug>/plan.md

# Read the ADR (Architect output)
cat Docs/design/adr/<N>-<title>.md
```

### 4. Track merged results
Once the squad completes, results are merged to `main` and the run directory becomes the historical record.

## 🛠️ Troubleshooting

**"Command not found: /agent squad"**
→ Ensure the squad plugin is installed: `copilot plugin list`

**"TypeError: a.replace is not a function"**
→ Known Copilot CLI v1.0.34 bug. Try upgrading: `copilot upgrade`

**"Git worktree already exists"**
→ Clean up: `git worktree list && git worktree remove .worktrees/<name>`

**Squad is stuck**
→ Check the reasoning log: `tail -50 Docs/squad/runs/<timestamp>-<task-slug>/reasoning.log.md`
